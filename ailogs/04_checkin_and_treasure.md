# 04 - Check-in & Treasure System

## Primary Prompt A
Upgrade check-in from mock scan to location-based check-in:
- User taps check-in button.
- System validates whether user is inside campus boundary.
- Daily limit: one successful check-in per day.
- Success grants one virtual collectible.
- Failure states must show clear feedback (already checked in / outside area / location unavailable).

## Primary Prompt B
Treasure design:
- Only two themes: **Captain Bird** and **XJTLU Bear**.
- Rarity rates: **Blue 70% / Purple 20% / Gold 10%**.
- Use provided image assets.
- Support bilingual naming and display in both check-in result and backpack.

## Primary Prompt C
Integrate check-in progress with badge logic:
- "Check-in Pro" should unlock after 3 valid check-ins.
